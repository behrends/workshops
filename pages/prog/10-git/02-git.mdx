# Git

Bisher haben wir ein Repository auf GitHub erstellt und 
Teammitglieder hinzugefügt. Nun geht es darum, die Dateien
auf den eigenen Rechner zu holen, Änderungen durchzuführen
und diese wieder auf GitHub zu speichern. Dazu verwenden
wir das Tool `git` im Terminal und VS Code.

Folgende Schritte führen wir gemeinsam durch:

- git auf dem eigenen Rechner installieren
- Projekt auf den eigenen Rechner klonen
- Änderungen an Dateien durchführen


## git auf dem eigenen Rechner installieren

Wer mit macOS arbeitet, hat `git` bereits auf dem Rechner
installiert. Meistens ist dies auch bei Linux der Fall. 
Für Windows müssen wir `git` herunterladen und installieren: 
https://git-scm.com


<Callout type="warning">
In die Runde fragen, wer `git` bereits 
installiert hat und was unter Windows zu
beachten ist. 

Im Installationsvorgang folgendes wählen: 

- VS Code als Editor 
- ansonsten die Vorschläge übernehmen
- Wichtig ist die Auswahl des `Git Credential Manager` für die Speicherung der Zugangsdaten.

Hier ist ein kurzes YouTube-Video, das die Installation auf Windows zeigt: https://www.youtube.com/watch?v=pfsRe7gKdrQ
</Callout>

Um zu prüfen, ob `git` bereits installiert ist, öffnen wir
das Terminal z.B. in VS Code (`View > Terminal`) und geben 
folgenden Befehl ein:

```bash
git --version
```

Dies sollte eine Versionsnummer ausgeben (`git version …`). Sollte eine 
Fehlermeldung erscheinen, dann ist `git` noch nicht installiert. 

&rarr; Zeit geben für die Installation von `git`.

## Git lokal konfigurieren

Zu Beginn nehmen wir ein paar Einstellungen 
in `git` vor. Wir teilen Git unseren Namen 
und unsere E-Mail-Adresse mit, damit in Git 
und GitHub nachvollzogen werden kann, wer 
eine bestimmte Änderung vorgenommen hat. Für 
die E-Mail-Adresse stellt uns GitHub eine 
`noreply`-Adresse zur Verfügung, die wir
verwenden können.

Dazu geben wir im Terminal folgenden Befehl 
ein &mdash; `USERNAME` durch den eigenen
GitHub-Namen ersetzen:

```bash
git config --global user.email "USERNAME@users.noreply.github.com"
```

Außerdem sollten wir unseren Namen angeben,
damit die Änderungen leichter zuzuordnen 
sind.

<Callout>
Wenn die Angabe des Namens aufgrund von 
Datenschutz nicht erfolgen soll, dann kann 
dieser z.B. durch ein Pseudonym ersetzt 
werden.
</Callout>

```bash
git config --global user.name "Max Mustermann"
```

## GitHub-Projekt auf dem eigenen Rechner einrichten

Wir wollen nun das Projekt von GitHub auf den eigenen Rechner
holen. Dazu verwenden wir das Terminal in VS Code.

<Callout>
Die sollte auch direkt aus VS Code möglich 
sein mit Authentifizierung via Browser 
&mdash; wichtig ist, dass die 
Zugangsdaten im `Credential Manager` 
gespeichert werden, damit wir nicht jedes
Mal die Zugangsdaten eingeben müssen.
</Callout>

In VS Code wählen wir zunächst mit `File > Open Folder...` den
Ordner aus, in dem wir das Projekt speichern möchten. 
Anschließend können wir das Terminal öffnen (`View > Terminal`)
und folgenden Befehl eingeben:

```bash
git clone PROJEKT-URL
```

Hierbei muss `PROJEKT-URL` durch die URL des 
GitHub-Projekts ersetzt werden. Diese finden 
wir auf der GitHub-Seite des Projekts, indem
wir den Button `Code` klicken und dann den
Bereich `HTTPS` wählen. Die URL beginnt mit
`https://github.com…` und endet mit `.git`,
z.B.:

```
https://github.com/USERNAME/PROJECT.git
```

`USERNAME` ist der GitHub-Nutzername und 
`PROJECT` ist der Name des Projekts.

<Callout type="warning">
Wir werden nun nach dem GitHub-Nutzernamen
und dem Passwort gefragt. Wenn wir diese 
nicht jedes Mal eingeben möchten, können wir
unsere Zugangsdaten im `Credential Manager`
speichern. Dazu sollten wir in Windows und
Mac automatisch gefragt werden.

In Linux kann das in jeder Distribution
etwas anders sein…
</Callout>

In der Dokumentation von GitHub wird die 
Speicherung der Zugangsdaten beschrieben:
https://docs.github.com/en/get-started/getting-started-with-git/caching-your-github-credentials-in-git

&rarr; Zeit geben für das Klonen des Projekts.

## Projekt in VS Code öffnen

Nachdem das Projekt geklont wurde, können wir
es in VS Code öffnen. Dazu wählen wir mit
`File > Open Folder...` den Projektordner 
aus (der eben geklonte Ordner) und öffnen
diesen.
