Browser-Tools in KI-Chats (Vibe)

Viele KI-Chats können Code nicht nur generieren, sondern auch direkt in einer isolierten Umgebung (Sandbox) ausführen.

Von kleinen UI-Komponenten oder Code-Schnipseln bis zu Projekten mit mehreren Dateien ist einiges möglich.

Browser-Tools in KI-Chats (Vibe)

IDEs für Vibe Coding

Entwicklungsumgebungen (IDEs) bieten neben dem Editor oft zusätzliche KI-Funktionen.

AI Assisted Engineering

Im Gegensatz zu reinem Vibe Coding bezeichnen wir „professionelle Zusammenarbeit“ mit der KI als AI Assisted Engineering.

Simon Willison, Mitbegründer des Web-Frameworks Django und bekannter KI-Blogger, schlägt den Begriff „Vibe Engineering“ vor:

I feel like vibe coding is pretty well established now as covering the fast, loose and irresponsible way of building software with AI. What should we call the other end of the spectrum, where seasoned professionals accelerate their work? I propose we call this vibe engineering
Quelle (oben leicht angepasst): https://simonwillison.net/2025/Oct/7/vibe-engineering/
Anders als bei Vibe Coding gibt es keinen festen Begriff — „AI Assisted Engineering“ ist eine mögliche Bezeichnung.

CLIs im Terminal als Assistenten

In Terminal-CLIs gezielt und planvoll mit der KI zusammenarbeiten:

Onboarding: „Analysiere dieses Repository und erkläre mir Architektur und zentrale Konzepte des Projekts.“

Qualität: „Untersuche den Code auf mögliche Bugs, Sicherheitsprobleme und offensichtliche Code Smells.“

Migrationen/Updates: „Erstelle einen Plan und Patches, um Bibliotheken und Abhängigkeiten zu aktualisieren.“

Systematisches Coding: „Hier sind Anforderungen an eine Funktion. Entwerfe passende Unit-Tests, ohne die Funktion zu implementieren.“

→ Typische Tätigkeiten eines Software Engineers → hier mit KI

Fähigkeiten von KI in IDEs

Tab-Completion: Code-Vervollständigungen durch KI beim Tippen

Chat im Editor: Fragen direkt inline zu bestimmtem Code stellen.

Plan- und Agent-Modus im Chat: komplexe Aufgaben in Schritte zerlegen und gezielte Code-Änderungen vorschlagen lassen.

Multi-Agenten (Agent HQ): spezialisierte Agenten (z.B. für Tests, Refactoring, Doku) arbeiten koordiniert am Projekt.

→ Aus der IDE lassen sich Aufgaben direkt an Cloud-Agents (z.B. GitHub oder OpenAI/Codex) delegieren.

Coding-Agents in der Cloud

Cloud-Agents laufen als Dienste in der Cloud, arbeiten eigenständig mit ganzen Repositories (z.B. GitHub), werden im Browser gesteuert.

Sie können große Codebasen analysieren, Änderungen planen und schrittweise umsetzen (Branches, Commits, Pull Requests).

IDEs und CLIs delegieren Aufgaben an diese Agents — der Agent übernimmt das „Heavy Lifting“ und meldet den Fortschritt zurück.

→ Eher für größere Änderungen, Migrationen und lang laufende Aufgaben geeignet (z.T. mehrere Stunden, perspektivisch länger).

Beispiele für Coding-Agents

OpenAI Codex : cloudbasierter Coding-Agent, der Repositories in einer Sandbox klont und Pull Requests vorschlägt.

GitHub Copilot Coding Agent : in GitHubs Cloud, verarbeitet Issues oder Pull Requests und erstellt bzw. aktualisiert passende PRs.

Weitere: Claude Code , Google Jules , Cursor Cloud Agents , …

Die Coding-Agents in der Cloud ähneln sich: sie klonen Repos, analysieren Code, planen Änderungen und liefern PRs zurück.

→ Coding Agents als „(Junior) Software Engineers in der Cloud“.

Fazit & Takeaways

1. Vibe Coding: schnell, explorativ, ideal für Ideen, Lernen, Experimente und Prototyping — „echte“ Aufgaben ggf. mit Risiko.

2. AI Assisted Engineering: professionelle Zusammenarbeit mit der KI planvoll, nachvollziehbar und reviewbar gestalten.

3. Drei Tool-Plattformen: Browser, IDE und Terminal bieten jeweils Vibe- und Engineering-Modi — passenden Kontext wählen.

4. Coding-Agents in der Cloud: arbeiten wie (Junior-)Engineers am Repository und liefern Pläne, Diffs und Pull Requests (viel Potenzial!).
