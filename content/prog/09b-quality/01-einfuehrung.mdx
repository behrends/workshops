# Einführung und Testpyramide

<Callout>
  **Dauer:** 30 Minuten

  - KI-Agenten schreiben zunehmend Code
  - Codequalität sicherstellen wird zur Kernkompetenz
  - Die Testpyramide als Orientierung

  **Ziel:** Motivation und Überblick über Werkzeuge zur Codequalität
</Callout>

## Motivation

KI-Agenten (z.B. GitHub Copilot, Claude, ChatGPT) schreiben
zunehmend Code. Dadurch wird es immer wichtiger, die Qualität
von Code systematisch sicherzustellen. Denn egal ob der Code
von Menschen oder von KI geschrieben wird &mdash; er muss
korrekt, lesbar und wartbar sein.

Codequalität umfasst verschiedene Aspekte:

- **Formatierung:** Einheitlicher, lesbarer Code
- **Typsicherheit:** Fehler durch falsche Datentypen vermeiden
- **Linting:** Häufige Fehlerquellen und schlechte Praktiken erkennen
- **Testing:** Sicherstellen, dass der Code korrekt funktioniert

## Die Testpyramide

Die Testpyramide ist ein Modell, das verschiedene Ebenen
der Qualitätssicherung von unten nach oben darstellt.
Weiter unten in der Pyramide sind die Maßnahmen
kostengünstiger und schneller durchzuführen.

```
        /\
       /  \
      / Ma \       Manuelles Testen
     /------\
    /  E2E   \     End-to-End-Tests
   /----------\
  / Integra-   \   Integrationstests
 / tionstests   \
/----------------\
|  Unit Tests    |  Unit Tests
|----------------|
|  Linting       |  Linting (ESLint)
|----------------|
| Statische Typen|  TypeScript
|----------------|
| Formatierung   |  Prettier
------------------
```

Die unteren Ebenen der Pyramide lassen sich leicht
automatisieren und geben schnelles Feedback.
Je höher man in der Pyramide steigt, desto aufwändiger
und langsamer werden die Maßnahmen.

## Unser Fokus

In dieser Vorlesung behandeln wir die unteren vier Ebenen
der Pyramide:

1. **Formatierung** mit [Prettier](/prog/09-quality/02-prettier)
2. **Statische Typen** mit [TypeScript](/prog/09-quality/03-typescript)
3. **Linting** mit [ESLint](/prog/09-quality/04-eslint)
4. **Unit Tests** mit dem [Node.js Test Runner](/prog/09-quality/05-unit-testing)

Diese Werkzeuge lassen sich in jedem JavaScript-/TypeScript-Projekt
einsetzen und bilden eine solide Grundlage für die Qualitätssicherung.

<Callout type="info">
Die höheren Ebenen der Pyramide (Integrationstests, E2E-Tests,
manuelles Testen) werden im [Ausblick](/prog/09-quality/06-ausblick)
kurz vorgestellt.
</Callout>
