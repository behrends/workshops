# Debugging und Tools

## Android

Für das Debugging nativer Android-Apps gibt es verschiedene Tools und Vorgehensweisen:
- Logcat: Android Studio bietet ein integriertes Logcat-Fenster, in dem
  Log-Nachrichten der App angezeigt werden. Hier können Debug-Ausgaben
  mit `Log.d("TAG", "Nachricht")` in Kotlin/Java ausgegeben und analysiert werden.
- Debugger: Android Studio verfügt über einen integrierten Debugger, mit dem
  Breakpoints gesetzt und der Code schrittweise durchlaufen werden kann.
- Profiler: Android Studio bietet Profiler-Tools zur Analyse der
  Performance, Speicher- und Netzwerkaktivität der App.
- Emulator-Tools: Der Android-Emulator bietet verschiedene Tools zum
  Testen und Debuggen, z.B. das Simulieren von Standortänderungen,
  Netzwerkbedingungen, etc. 

## React Native

Für das Debugging von React Native Apps gibt es ebenfalls verschiedene Tools:
- React Native Debugger: Ein eigenständiges Tool, das den Chrome DevTools
  ähnelt und speziell für React Native entwickelt wurde. Es bietet
  Funktionen wie Inspektion des Redux-Stores, Netzwerk-Analyse, etc.
- Expo DevTools: Wenn die App mit Expo entwickelt wird, bietet Expo
